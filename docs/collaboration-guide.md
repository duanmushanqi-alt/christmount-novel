### 三、第一卷核心经验总结（10条）

这是第一卷写完后最值得记录的规律，后期工作的出发点：

1. **追踪系统必须全穿透**：每章写完立刻更新4个JSON，哪怕只改几行。延迟更新会导致后续章节出现"角色提前知道不该知道的信息"。

2. **任务规格的"必须包含"保底**：tasks.md里每章的"必须包含"列表，是防止内容空洞的底线——写完后对照一遍比事后修改省力。

3. **writing-style-guide.md 是活文档**：每当写出一种新技法（如第1-3章的"外松内紧"，第7-15章的"情绪克制型"），应立即归档到本文件对应章节，否则会遗忘或不一致。

4. **character-state.json 的"知识边界"是悬疑类故事的命脉**：黄简在第X章才能知道的事情，不能在第X-1章的叙述层面透露。每次写前必查知识列表。

5. **对话"信息密度"优先于"情绪密度"**：一段对话如果只有情绪/气氛而没有信息量，读者会察觉拖沓。黄简的语言极简风格（"继续说。"）是其知识本质，不能为了"有趣"而让他变啰嗦。

6. **每章"钩子"比章节本身更重要**：第一卷节奏好的章节，其章末悬念几乎都在tasks.md里预先设定。没有预设钩子时，容易收尾无力。

7. **场景切换用"---"分隔不可省略**：视角切换或时间跳跃没有分隔符时，读者体验混乱。这是第一卷多次核查后确认的硬规则。

8. **AI不主动修改规格文档**：所有规格变更（改chapter大纲/改角色性格定位）必须由人类决策后，AI执行落地。AI提建议可以，但不单方面更新specification.md。

9. **模糊记忆用/clarify消除**：每次新章节开写前，如果对当前章节任务的方向有任何模糊，用/clarify问清楚比写错再改省力。

10. **季节/时辰/地点是一致性最高频失误点**：第一卷已将季节（秋末冬初）和地点（兴城城内/城外/兴城郊外）的写法收录进writing-style-guide.md第十四节。每卷开始需要明确本卷的时间段和主场景地图。

## PART B: 协作流程详解

### 一、新卷启动 SOP

在写第一章之前，完成以下检查清单：

#### 阶段1：确认规格基础（人类主导）

- [ ] 读 specification.md，确认本卷的主题和成功标准
- [ ] 读 creative-plan.md，确认本卷章节数、主要情节弧
- [ ] 明确本卷的**时间跨度**（例：第一卷=逃狱后13天）
- [ ] 明确本卷的**主要场景地图**（例：第二卷=藏宝地◇西蜀境内）
- [ ] 确认**主角当前状态**（读 character-state.json 最后一行）
- [ ] 确认**未收回的伏笔清单**（读 plot-tracker.json 中 future_hooks）

#### 阶段2：拆解任务（AI协助）

- [ ] 使用 `/tasks` 命令，基于本卷大纲生成章节任务清单
- [ ] 人工审核每章"必须包含"列表是否完整
- [ ] 确认每章的"涉及线索"是否与 plot-tracker.json 中的线索ID对应
- [ ] 更新 tasks.md

#### 阶段3：初始化追踪（AI执行）

- [ ] 使用 `/track-init` 命令，以当前追踪文件状态为基础，添加本卷预期事件节点
- [ ] 验证 timeline.json 中最后一个已完成事件的时间与本卷起点衔接无误

#### 阶段4：开写前最终确认（人类决策）

- [ ] terminology.md 是否需要补充新词（新地名?新人物?新物品?）
- [ ] writing-style-guide.md 是否需要补充本卷特有的写作规则（新场景类型?）
- [ ] 向AI下达本卷的"额外约束"（如：第二卷黄简身份变为西蜀贵族，语气微调方向）

### 二、章节写作循环（六步 Loop）

```
Step 1  ──►  Step 2  ──►  Step 3  ──►  Step 4  ──►  Step 5  ──►  Step 6
确认任务      读入上下文     AI写作       人工审核      更新追踪      可选验证
  │             │            │            │             │            │
tasks.md     前章末尾      /write       检查语气      /track      /checklist
本章规格     tracking     完整输出      逻辑漏洞      四个JSON     规格对照
             文件当状态               角色知识边界
```

#### Step 1 — 确认本章任务

在 tasks.md 中找到本章任务条目，确认以下内容已读清楚：
- 核心任务（这章要完成什么）
- 关键情节（必须写到的具体情节点）
- 必须包含（强制元素，缺一不可）
- 伏笔埋设（本章要种下哪个伏笔）
- 章末钩子（这章结尾留什么悬念）
- 依赖（确认上一章的输出已写完）

#### Step 2 — 读入上下文

AI在写作前必须加载：

| 文件 | 用途 |
|------|------|
| 上一章末尾200行 | 叙事语气、场景延续、末尾钩子承接 |
| `character-state.json` 主角块 | 当前位置、心理状态、知识边界 |
| `timeline.json` 最后事件 | 当前故事时间点 |
| `relationships.json` 本章相关角色 | 关系动态，对话基调 |
| `terminology.md` | 本章出现的地名/称谓 |

#### Step 3 — 执行写作（/write）

AI输出完整章节，格式标准：
- 文件头部必须包含：章节编号、章节标题、字数、核心任务简述
- 每个场景之间用 `---` 分隔
- 时间推进必须用古代时辰标记（如：卯时、辰时、午时）
- 结尾必须包含钩子或伏笔

#### Step 4 — 人工审核（必经环节）

人类重点检查以下项目（按重要程度排序）：

1. **角色语气是否OOC**：黄简是否变啰嗦了？林玄影是否过于清醒？
2. **角色知识边界**：这章有没有角色"提前知道"还没揭露的信息？
3. **时间线是否合理**：情节发展所需时间与上一章末尾衔接是否自然？
4. **关键情节是否遗漏**："必须包含"列表逐项对照
5. **AI痕迹**：有无出现堆叠式描写（"一丝XX，一丝XX，还有一丝XX"）

> 若发现问题：用自然语言告知AI具体问题，AI定点修改，不重写整章

#### Step 5 — 更新追踪文件（/track）

每章写完后必须更新：

| 文件 | 更新内容 |
|------|---------|
| `timeline.json` | 新增本章时间事件节点，故事时间推进 |
| `character-state.json` | 更新主角/涉及角色的位置、心理状态、知识列表 |
| `plot-tracker.json` | 标记本章完成的情节节点，新增触发的伏笔 |
| `relationships.json` | 记录本章发生的关系变化 |

#### Step 6 — 质量验证（/checklist，建议每3-5章或卷末执行一次）

执行 `/checklist` 命令，自动检查：
- 规格符合度（与specification.md核对）
- 内容扫描（AI痕迹、情绪标签化、时间矛盾）
- 术语一致性（对照terminology.md）

### 三、协作角色分工

#### 人类的核心职责

| 职责 | 具体内容 |
|------|---------|
| **方向决策** | 故事大纲走向、角色命运、情节是否合适 |
| **角色主权** | 最终判断"这像不像这个角色说的话" |
| **规格变更** | 任何对specification/creative-plan的修改，人类决策后AI落地 |
| **审核把关** | 每章完成后的人工审核（Step 4）|
| **风格感知** | 哪些地方"感觉不对"——人类的直觉是最终判断标准 |

#### AI的核心职责

| 职责 | 具体内容 |
|------|---------|
| **内容生成** | 执行写作，按tasks.md规格输出完整章节 |
| **上下文维护** | 确保每章写作前加载必要的上下文文件 |
| **追踪文件更新** | 写完每章后更新4个JSON追踪文件 |
| **规范执行** | 严格遵守writing-style-guide.md，不自行引入新风格 |
| **知识边界守护** | 主动检查角色的知识边界，防止信息泄露 |
| **建议而非决策** | 有改动建议时以"建议"形式提出，最终由人类拍板 |

#### 暂停信号（人类的优先级最高）

以下情形，人类应明确暂停写作流程，先澄清再继续：

- 角色行为感觉与人设不符（即使"逻辑上说得通"）
- 某个情节设计有更好的可能，想重新选择方向
- 追踪文件和文章内容之间出现矛盾无法调和
- 发现之前章节有需要修改的连锁错误

## PART C: 写作范式库

以下范式均来自长篇连载小说的实际写作经验，每个都附有示例以便对照。

---

### 范式1：极端处境——"外松内紧"双层叙事

**适用场景**：逃脱、躲藏、被埋、高危险场景

**核心原理**：主角外表动作冷静甚至缓慢，内心时钟高速运转。两层形成反差，张力远高于直接写"他非常紧张"。

**结构模板**：
```
[外层——小动作，精确，几乎无感情色彩]
他开始脱老者的衣服。布料很旧，边缘磨到发毛。他尽量小心，像是怕弄疼老人一样。

[内层——生理反应，暗示极度紧绷]
草席下一片漆黑。黄简能听到自己的心跳，一下接一下，像是要从胸腔里跳出来。

[威胁升级——外部声音或危险逼近]
外面传来脚步声。一步，两步。停在草席边。

[假解除——读者刚松一口气]
"行了，走吧。"

[真威胁出现]
"等等——你看那儿……"

[最终解除——才能移动]
良久，脚步声消失在远处。[主角]才敢轻轻呼出一口气。
```

**配套技法**：
- 被埋/躲藏时，以**听觉+触觉+嗅觉**替代视觉叙事
- 等待间隙（而非动作高潮）是插入内心独白的时机
- 危险方的对话不能是废话，在闲聊中顺带推进对主角有用的背景信息

---

### 范式2：高情绪场景——克制型爆发

**适用场景**：愤怒、悲痛、内疚，凡是强烈情绪的场景

**核心原理**：**不靠内心独白，靠身体动作**。延迟的情绪比即时爆发更有力。

#### 2A — 压抑型愤怒

```
✅ 拳头落下，骨节发白，没有说一个字。
✅ 沉默比骂出口更沉重。
❌ "他想起了多年的屈辱，愤怒如熔岩涌出……"
```

爆发结构：
```
1. 触发点（对方说了一句话/做了一件事）
2. [停顿——主角什么都没说]
3. 身体动作（拳头、移步、抓住）
4. 爆发（可以是暴力，可以是极少的几个词）
5. 爆发后的空白/脱力——比爆发本身更重要
6. 另一方的反应（[对方不还手/不回击] > 千言万语）
```

#### 2B — 内疚/自责

```
✅ "都是我的错……害了你……害了所有人……"（语无伦次，碎片化）
❌ "当年我做了那件事，导致了你……"（太清醒，太完整）
```

外化行为比独白更有力：
- [具有内疚心理的角色]反复做某个无意义的动作（如：一遍遍画从未画完的同一幅画）
- 从主角/旁观者的**外部视角**看[内疚角色]，比直接写内心更具冲击

#### 2C — 悲痛的克制

```
延迟序列：
1. 触发点出现（见到某物/某地/某人）
2. 先描写控制（手指收紧，将[物品]握得更紧了一些）
3. 然后描写失控（才许可哭泣）
4. 哭泣之后的沉默 > 哭泣本身
```

---

### 范式3：长对话场景——打断式叙事

**适用场景**：两人交代信息量大的对话

**核心问题**：超过1000字的对话容易形成"对白墙"，读者视觉疲劳。

**解决方案**：打断式叙事

```
[对话片段1——释放信息A（2-3轮）]
"[重要信息A]"
"……"

[打断：环境细节/时间流逝]
天边开始泛白。蜡烛烧短了一截。

[对话片段2——释放信息B（2-3轮）]
"[旧事/关键线索]……"

[打断：角色动作]
[主角]移开了视线。

[对话片段3——释放信息C]
```

**规则**：
- 每隔2-3轮对话，插入一句环境细节或动作
- 重要信息说完后，必须留一段沉默/过渡，再继续
- 一次对话只讲清楚**一件事**，不要让角色一口气说完所有

---

### 范式4：信息揭露场景——知情差设计

**适用场景**：读者和主角同时拼凑真相

**核心工具**：三层知情差

| 层次 | 知道什么 |
|------|---------|
| 读者 | 看到主角的全盘计划和推理过程 |
| 主角 | 掌握碎片线索，正在推理 |
| 配角 | 只看到局部表现，不知道主角的全貌 |

**推理展示结构**：
```
[观察到一个细节——用短句单独成段，让读者注意]
[主角]注意到一个细节。

[提问——邀请读者思考]
为什么？

[细化描写——给读者自己推理的材料]
[描写这个细节的具体规律：谁在做什么、有什么差异]

[主角得出结论]
[主角]仔细观察了一会儿，明白了。

[解释逻辑——简洁，2-3句]
[原因分析]

[行动意向]
这就是突破口。
```

### 范式5：章节任务规格模板

这是 tasks.md 每章条目的标准格式，新卷任务拆解时使用：

```markdown
#### 第X章：[标题]

- [ ] [P0] **TXXX** - 第X章：[标题] ([预计字数])
  - **核心任务**：[一句话：这章要完成什么核心目标]
  - **关键情节**：
    - [情节点1：具体场景/事件]
    - [情节点2]
    - [情节点3]
  - **涉及线索**：
    - PL-01(主复仇线) ⭐⭐⭐ [对主线的贡献]
    - PL-0X([线索名]) ⭐⭐ [贡献]
  - **必须包含**：
    - [要素1：此章不写则内容缺失的关键要素]
    - [要素2]
  - **伏笔埋设**：
    - F-XXX：[伏笔内容]
  - **章末钩子**：[结尾留下的悬念或转折]
  - **依赖**：TXXX（上一章编号）
  - **输出**：`stories/content/volumeX/chapter-XXX.md`
```

### 范式6：章末伏笔/视角切换模板

第一卷中，多个章节结尾用"另一方视角"埋下伏笔，这个结构效果最好：

```markdown
---

与此同时，[另一地点]，[另一人物]，

[场景设定——简短，1-2句]

[人物动作或话语——携带信息量或悬念]

[画面定格——看向某处，夜色，不知答案]
```

**示例**：
```
与此同时，[异地城市]，一座幽静的宅邸内，

书房刚刚点起蜡烛，小厮走进屋内低声禀报："少爷，查了两年了，还是没什么消息。要不……"

"继续查。"[另一角色]打断他，声音很轻，但不容置疑。

书房里重新安静下来。[另一角色]看向窗外，夜色深沉，看不到尽头。
```

### 二、一致性最高频失误点（第一卷审计结果）

| 失误类型 | 具体问题 | 验证方法 |
|---------|---------|---------|
| 季节错误 | 本卷时间段与章节内气候描写矛盾 | 写前确认本卷时间段 |
| 时辰矛盾 | 同一夜晚时间顺序前后不符 | 检查 timeline.json |
| 称谓混用 | 同一人物在不同章节有多个称谓写法 | 对照 terminology.md |
| 地点混淆 | 城内/城外/郊外场景边界混用 | 写前明确场景地图 |
| 角色超前感知 | 角色引用了当前章节才揭露的信息 | 对照 character-state.json |
| 专有名词混淆 | 多势力/多国设定的正式名称写错 | 搜索 terminology.md |

## APPENDIX: 写作范例

> 精选能体现各范式的段落，供写作对照

### 范例A：极端处境——躲藏/被埋场景

```
他开始脱老者的衣服。布料很旧，边缘磨到发毛……他尽量小心，像是怕弄疼老人一样。

草席下一片漆黑。[主角]能听到自己的心跳，一下接一下，像是要从胸腔里跳出来。

外面传来说话声：
"……新上司那边怎么说？"
"还能怎么说，找不到就烧了。都一样。"

土一铲铲落下来。声音越来越沉，越来越近。
```

> **范式标注**：外层动作（脱衣服）→内层紧绷（听到心跳）→外部威胁（铲声）→威胁对话顺带推进信息→等待

---

### 范例B：极简对话——高克制型主角的标准语气

```
"继续说。"
"……"
"嗯。"
"我知道了。"
```

> **范式标注**：每句话有且仅有必要的信息量。沉默（"……"）是此类主角最常见的"回应"。话越少，分量越重。

---

### 范例C：推理结构——观察侦查场景

```
[主角]注意到一个细节。

那些穿着朴素、挑着本地蔬菜的农民，守卫往往只是瞄一眼，就挥手放行了。

为什么？

[主角]仔细观察了一会儿，明白了。

这些农民，大多衣着简陋，手上、脸上都是泥，挑着担子或推着车，一看就是刚从地里来的。本地人，干苦活的，没什么油水，也不会是什么危险人物。守卫懒得仔细查。

这就是突破口。
```

> **范式标注**：观察→提问→细化描写→主角得出结论→行动意向。节奏紧凑，读者跟着主角一起推理。

---

### 范例D：克制型悲痛——情感高峰场景

```
手指收紧，将香握得更紧了一些。

他没有说话。

那是[主角]记事以来，第一次在父母墓前落泪。
```

> **范式标注**：小动作（收紧）→沉默→延迟爆发（第一次落泪）。"第一次"的标注给这个瞬间盖章，胜过大段铺陈。

---

### 范例E：宣言式结尾——卷末收束场景

```
"从今往后——[主角旧名]，已经死了。死在多年前那条山道上。"

他拍了拍[旧友]的肩，像是在做某种仪式性的告别。

然后他转身，走出了[旧友居所]的大门。

[故乡城市]的冬日天光逐渐明亮，但已经和他没什么关系了。
```

> **范式标注**：宣言（旧身份终结）→仪式性动作→转身离开→环境收尾（"和他没什么关系了"的疏离感）。卷末标准收束结构。
