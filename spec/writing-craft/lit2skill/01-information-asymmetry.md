# Skill 1：信息差叙事

> 本作系列 lit2skill 改造版第 1 号技法。主要借鉴来源：《琅琊榜》。

## description（A2 未来触发场景）

写"化名主角"与前身旧识互动 / 卧底角色在敌人中生存 / 身份秘密类布局故事 / 陌生人偶遇但读者已知主角身份。当故事依赖"读者比角色多知道一层"产生张力时调用。

## I 骨架（可操作定义）

让读者知道的信息集合 ⊋ 场景内任一角色知道的信息集合。每个动作、台词对场景内角色成立一层含义，对读者成立另一层。张力不来自情节推进，来自"读者看见角色没看见的那层"。

## A1 可对照的书中/本作场景

- 琅琊榜：梅长苏 × 靖王重逢 / 梅长苏 × 霓凰 / 梅长苏 × 谢玉交锋
- 本作第 16 章：黄繁 × 茶肆赶车人（读者知道他刚出牢、揣玉牌、肩上有仇）

## E 可执行步骤

- **E0 前置判断**：读者此刻是否知道角色不知道的关键信息？（没有 = 不调用本 skill）
- **E1 建立信息差**：化名/真身一类的核心信息必须在**本场戏之前**向读者交代清楚，不能留到事后揭晓才回溯张力
- **E2 双清单**：写每场外部互动前，列两份清单
  - 场景中角色 A/B/C 看到听到的（表面层）
  - 读者独有的信息（暗面层）
- **E3 表里分离**：主角台词做到**表面成立**（对场景内角色合理自洽）、**暗面有弦**（对读者至少有一处钩）
- **E4 分级管控揭晓**（v1.1 三色分级）：
  - 🔴 禁用：主角视角直接自白（"他心想：我其实是……"）→ 爆破信息差
  - 🟡 节制：暗示式注解（例如"这是他头一回对陌生人报这个身份"）→ 不暴露真身，每场不超 1 处
  - 🟢 推荐：纯场景叙事，信息差让读者自己读
- **E5 场景收尾不自解说**：信息差继续悬着进入下一场

## 诊断钩子（写完扫一遍）

1. 这场戏**删掉信息差**是否完全成立？（完全成立 = 没用上技法，当前场景性价比存疑）
2. 读者多知道的那层，**每句关键台词**都有意识利用了吗？
3. 主角视角出现"他心想：其实……"？→ 🔴 红线，删
4. 场景内有 >1 处黄线注解？→ 压缩到 ≤1 处

## B 边界与盲点

- 全员上帝视角的故事用不上
- 揭晓过早张力立塌——本作第 1 卷前 3 章已向读者交代身份，第 2 卷起全程可用
- 滥用 → "全员低能综合征"。必须用**环境闭塞 / 角色粗疏 / 伪装合理**三选一护住前提
- 长期维持 → 主角显得"算无遗策"。需用**险情 / 破绽 / 被怀疑**平衡（参考本作第 16 章茶肆复盘段落）

## 正反对照

- 平庸：主角自白"我就是林殊"→ 直给，信息差爆破
- 本作第 16 章：黄繁说"陈汉国""行脚的""西边"→ 对赶车人是普通商旅信息，对读者是伪装破绽的放大镜

## 与 pitfalls 交叉索引

- 违反 E4 🔴 ≈ P-15（动机描写过度解释，把内心链条摆给读者看）
- 违反 E1 → 常表现为 P-01（全知视角污染角色反应，读者还没建立信息差就让角色按"已知"反应）
